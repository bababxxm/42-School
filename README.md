## üåê Environment

42 School is a **peer-driven, self-paced coding school**. Every project is evaluated within a structured environment to ensure learning, skill growth, and code quality.

---

### üìù Norminette (Coding Standard)
Almost every project must follow the **Norm**, the official coding standard. This ensures **readable, maintainable, and professional code**.

**Key Rules:**
- Maximum **25 lines per function** (excluding braces)
- Maximum **80 characters per line** (including comments)
- Maximum **4 parameters per function**
- Maximum **5 variables per function**
- No forbidden constructs: `for`, `do...while`, `switch`, `case`, `goto`, `ternary operator`
- No assignments on the same line as declaration (unless `const` or `static`)
- Mandatory **function prototypes and headers**

üí° Following the Norm is mandatory for **peer evaluations** and for passing the **Moulinette** checks.

---

### ü§ñ Moulinette (Automated Evaluator)
**Moulinette** is 42 School‚Äôs automatic grading system.  
It checks your code for:

- Norm compliance ‚úÖ
- Correct functionality against project specs üõ†Ô∏è
- Memory errors and leaks (C projects) üíæ
- Output format and edge case handling üìÑ

**Evaluation Types:**
| Type | Who Evaluates |
| :--- | :----------- |
| Solo project | Moulinette or Peer |
| Group project | Peers, sometimes verified by Moulinette |
| Exams | Moulinette only, time-limited (2‚Äì4 hours) |

---

### üìä Evaluation & Progress
Projects and exams use a **points system** (usually out of 100 or 125).  
Additional details:

- **Peer Evaluations Required:** Some projects need multiple peers to approve before completion.
- **Bonus Points:** Extra features or optimizations can give bonus points.
- **Blackhole Days:** Completing projects can grant extra days to postpone future deadlines.
- **Passing Requirements:** A total number of evaluation points is needed to progress through the Common Core.

üí° **Tip:** Always submit code that passes the Norm and is peer-ready. Peer evaluations can influence your ability to move forward.

---

Understanding **Norm**, submitting projects to **Moulinette**, and completing **peer evaluations** is key to progressing smoothly at 42 School. Projects combine **individual work** with **collaboration**, teaching both technical and soft skills.

## PISCINE

The **Piscine** (French for "swimming pool") is a **4-week intensive bootcamp** at 42 School designed to introduce you to programming fundamentals.

- Focused on **C programming**, problem-solving, and algorithmic thinking.
- Projects are **solo and hands-on**, emphasizing practical skills over theory.
- Students are **immersed in coding**, often working long hours to complete exercises.
- Goal: Build a strong foundation before tackling more advanced projects in the **Common Core**.

üí° Think of it as the **foundation layer** of your 42 School journey.

---

## COMMON-CORE

After the Piscine, you progress to the **Common Core**, which includes **mandatory and bonus projects** organized by ranks.

- **Rank Structure:** Projects are grouped by rank (e.g., Rank 00 ‚Üí Rank 06). Each rank usually has **3‚Äì5 projects**.
- **Exams:** Each rank may include an **exam project**, evaluated only by the **Moulinette**. Exams test your understanding of the rank‚Äôs core concepts.
- **Solo vs. Group:** Some projects are **solo** (e.g., Libft, Pipex), others are **group projects** (e.g., Minishell, Cub3D). Group projects teach collaboration and code integration.
- **Languages & Tools:** Covers **C, C++, Web, Networking, and Docker**, depending on the rank and project.
- **Evaluation:** Each project has a **point grade**, sometimes requires **peer evaluations**, and may grant **bonus points** for extra features.

üí° The **Common Core** is where you **apply the fundamentals from the Piscine to more complex, real-world projects**, gradually increasing in difficulty and scope.

## ADVANCED CORE

After completing the Common Core, students move on to the **Advanced Core**, which focuses on specialized and high-level projects. These projects aim to prepare students for **real-world software development**, combining multiple skills learned in previous ranks.

- **Specialized Projects:** Covers advanced topics such as **full-stack web development, complex C++ applications, networking, and deployment using Docker or cloud technologies**.
- **Group Collaboration:** Most projects are **group-based**, simulating real-world team development with multiple contributors, code reviews, and project management.
- **Exams & Evaluation:** Some projects include **exam-like assessments** evaluated by the Moulinette to test mastery of the concepts. Peer evaluation may also be required.
- **Languages & Tools:** Students apply **C, C++, Web technologies, Networking, Docker, NestJS, PostgreSQL, OAuth**, and more depending on the project.
- **Goal:** Prepare students for professional development by integrating **software engineering best practices, testing, deployment, and teamwork**.

üí° Think of the Advanced Core as the **capstone stage of your 42 School journey**, where you consolidate your skills, tackle challenging projects, and gain experience closely aligned with industry expectations.
